# Architecture & Implementation

This document outlines the technical decisions, design patterns, and implementation details that power **RepoVis**.

## 1. Visual Identity & Layout
- **Decision:** Full-screen responsive grid with vibrant neon accents on a dark background.
- **Rationale:** Optimized for high-density information display. Neon accents against dark backgrounds (using layered CSS box-shadows for bloom) create a high-tech, "cybernetic" feel.
- **Implementation:** Uses a responsive CSS grid (`max-w-7xl mx-auto grid-cols-1 md:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4`).

## 2. Interaction Model: Progressive Disclosure
- **Decision:** Multi-layered expansion from Macro Grid -> Deep Dive -> Context Sub-Card.
- **Rationale:** Avoids overwhelming the user. The `MiniVis` provides ambient information at a glance. Single-clicking into a deep state allows for focused graph exploration. Clicking an intra-graph node renders a `ContextSubCard` panel with raw log traces inline securely.
- **Implementation:** Uses `framer-motion` layout groups (`layoutId`) for seamless shared-element transitions between the grid tile and the full-screen modal, and absolute overlay rendering for Sub-Cards.

## 3. Visualization Technology Stack
- **D3.js**: Used for complex physics and relationship modeling (e.g., the Gravity Well in FocusDeep, curved paths in RelationshipsDeep).
- **Framer Motion**: Handles layout transitions, route-like animations, and micro-interactions.
- **Pure CSS**: Used for high-performance ambient loops in the `MiniVis` components.
- **Web Audio API**: Serves a complex procedural synthesized diagnostic engine simulating real hardware UI states conditionally tracking artifact severity.

## 4. Analytical Engines & Parsing
- **AST Drift Parser (`/lib/analysis/drift.ts`):** Employs the `typescript` compiler API to construct Abstract Syntax Trees of repository scopes natively. Extracts relative/alias imports and applies RegEx-based rule matrices (e.g. `lib/` shouldn't call `components/ui/`) to determine if standard boundaries are maintained, throwing high priority Drift exhibit anomalies on failure.
- **Gravity Association Module (`/lib/analysis/gravity.ts`):** Parses raw internal commit histories mapping files altered within the exact same chronological hash, producing a statistical matrix mapping satellite dependencies normalized into a strictly 0.0 to 1.0 `coEditScore`.
- **API Runtime (`/app/api/analyze/route.ts`):** Acts as a bridge connector simulating file-system traversing against the app's own deployment workspace to emit live payload generation metrics.

## 4. Specific Exhibit Patterns

### Focus (Gravity Well)
- **Logic:** Visualizes branch and file activity.
- **Motion:** Satellite nodes orbit a central active file. Distance is proportional to `coEditScore`.
- **Encoding:** Center radius pulses with `commitFrequency`.

### Relationships (Force Graph)
- **Logic:** Links between PRs, Issues, and Files.
- **Motion:** Animated particles flow along connection lines to show direction.
- **Encoding:** Path curvature (Quadratic Bezier) encodes time since last interaction. Node size encodes `significance`.

### Drift (Architectural Integrity)
- **Logic:** Contradictions between documentation and code.
- **Motion:** "Tectonic shake" animation in MiniVis reflects severity.

### Churn & Snapshot
- **Logic:** File modification frequency and module blueprints.
- **Encoding:** Equalizer-style bars and grid-based heatmaps.

## 5. Agentic Narrative System
- **Concept:** RepoVis is a feed of artifacts generated by specialized agents (e.g., `Forecaster`, `ChurnAnalyzer`).
- **Implementation:** 
    - Each artifact has a `narrative` field providing a human-readable synthesis.
    - A "Whisper" layer provides raw metadata fallback in deep views.
    - A diagnostic sub-system in the footer tracks agent health and failures.

## 6. Performance & Constraints
- **Solo-Developer Context:** Optimized for a high-density, IDE-adjacent feel.
- **No-Referrer Policy:** All images use `referrerPolicy="no-referrer"` to ensure compatibility with restricted hosts.
- **Client-Side D3**: D3 simulations run entirely on the client for maximum responsiveness and interactivity.
